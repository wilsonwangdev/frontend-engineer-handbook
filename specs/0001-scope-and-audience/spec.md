## Context

`frontend-engineer-handbook` 要成为一本面向不同经验前端工程师的中文精编手册。
需要在动笔前先锁定范围，避免越写越像"中文百科聚合"。

参照对象：[roadmap.sh/frontend](https://roadmap.sh/frontend) 提供完整的可点开
路线图，但它（1）以英文为主，（2）AI 时代特性覆盖偏弱，（3）节点数量过多，
给人"永远学不完"的感觉，更接近"全景百科"而非面向实践的精编路线。

本手册的差异化：
- **做减法**：只记录必要信息，不追求覆盖面。
- **显式表达取舍判断**：告诉读者什么不用学、为什么。
- **填补中文优质站点的缺失**：不是翻译，是原生中文视角的判断与路线。
- **入口 + 速查 + 实践**：为读者提供快捷学习/复习入口，保留权威来源链接
  供深入考证，末尾附实践项目巩固理解。

## Decision

### 定位（载入性）

1. **时间锚点：面向 2026 年**。不为兼容历史而保留过时知识。任何 2020 年前
   就该淘汰的内容（浮动布局、CSS Hack、IE 兼容、ES5 polyfill 写法、Jest
   早期历史、Webpack 详尽配置等）一律不进主干。
2. **核心读者画像：围绕 agent 协作的前端工程师**。手册预设读者会在日常工作中
   把部分任务委托给 AI / coding agent，因此：
   - 知识点的"必要性"是以**人 + agent 协作**为前提评估的，不是单人工作。
   - 工程师价值的新落点是：**判断、上下文管理、验证、品味**——而非记忆 API
     与样板代码。
3. **小而美，不是大而全**。每章自洽，可"随时翻阅"。读者不应感到"永远
   学不完"——手册的职责是画出清晰边界，让读者知道学到哪里就够了。
4. **入口与速查定位**：手册本身提供精炼的知识点讲解与判断，同时保留权威
   来源链接（MDN、web.dev、框架官方文档等），供读者按自己的方式深入。
   手册不试图替代一手文档，而是做"导航 + 判断 + 快速复习"。
5. **面向实践**：末尾设置动手项目章节（mini bundler、mini framework、
   mini agent app），让读者通过构建来理解原理，而非停留在阅读。

### 读者分层（三档）

- 新入行 / 转行：需要清晰主干路线
- 1–3 年经验：查漏补缺，对标必要知识点
- 资深：作为速查与"AI 时代什么不再需要学"的判断参考

### AI 时代取舍的显式表达（双轨制）

1. **总览章主动列出"AI 可委托清单"**：明确告诉读者哪些任务现在已可以放心
   委托给 AI（如 Webpack 复杂配置、CSS 兼容性 Hack、正则调试、样板组件、
   commit message 等）。立场鲜明、有争议——但这是手册的核心价值之一。
2. **每个知识点旁标三档归位**：必学 / 仍需理解但不必精通 / 可委托给 AI。
   附带**简短理由**。
3. 两条轨道并存：总览给读者"地图"，章节内给读者"在地图上的具体位置"。

### 主干与附录的边界

- **主干**：必要学习路线 + 必要知识点 + 必要的现代化视野扩展 + 必要的工程
  之上的高级话题。
- **主流框架**：主干**仅**展开 React + Next.js（详见 [SPEC-0003](../0003-table-of-contents/spec.md)）。
- **附录**：Vue / Svelte / Solid 等其他框架对照、扩展方向（全栈 / 移动 /
  可视化）、术语对照、精选资源清单（含可选经典书目）。
- 优质外部资源仅作为延伸阅读附在章节末尾，不替代主干内容。手册保留
  权威来源链接，供读者按自己的方式深入考证或学习——手册是入口和导航，
  不试图替代一手文档。
- **实践章节**（第 10 章）：通过构建 mini bundler、mini framework、
  mini agent app 让读者从"读"升级为"做"。每个项目控制在一个周末可完成。

### 语言

简体中文为主。术语首次出现时附英文原文，方便检索一手资料。

## Consequences

- 写作时的高频判断变成"这个值不值得放进主干"，而不是"这个内容写得够不够全"。
  默认从严，宁缺毋滥。
- 任何章节都要回答四个问题：必学什么？为什么 AI 时代仍要学？三档归位如何？
  哪里能找到一手权威资料？
- 主干只走 React + Next.js 一条主线，会让 Vue 生态读者觉得不亲切——附录的
  框架对照必须做扎实才能弥补。
- "面向 2026 年"是一个会过期的承诺，需要在 SPEC-0002 的时效性条款下定期复核。

## Alternatives considered

- **做完整百科**：和 MDN、roadmap.sh 重复，价值低。
- **做单一线性教程**：无法服务三类读者；放弃。
- **React + Vue 双主干**：体量翻倍，且会稀释判断的鲜明性；放弃。
- **框架无关 + 框架进附录**：主干太抽象，新入行读者落地困难；放弃。
- **多语言**：分散精力；先做中文，未来可再议。
