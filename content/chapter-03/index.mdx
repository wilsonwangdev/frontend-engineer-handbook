## 一段必要的史前史：CSS 是怎么"长成这样"的

> 如果你觉得 CSS 哪里别扭——比如选择器优先级总要数权重、margin 会
> 神奇折叠、`float` 的清除要靠"伪元素 hack"——**那不是你的问题，
> 是这门语言本来就是从一片乱里收敛出来的**。知道它的来路，下面这章
> 的很多设计就不再"反直觉"。

90 年代初的 Web 没有"样式表"这个东西。HTML 自己兼任表现职责——
`<font color>`、`<center>`、`<bgcolor>` 这种**表现型标签**直接写
进结构里，加上 Netscape 和 IE 各自塞的私有属性，整个网页是
"结构 / 样式 / 行为"全部糊在一起的一团泥。

1996 年 W3C 发布 [CSS Level 1](https://www.w3.org/TR/CSS1/)，把"怎么
显示"从 HTML 里拆了出来。这是 Web 史上**最重要的一次分层**——从此
HTML 管"是什么"，CSS 管"长什么样"，JS 管"做什么"。这套分工是现在
所有前端框架的根基。

但 CSS 走得很慢：

| 年份      | 节点                                                                 | 留下了什么                                                                        |
| --------- | -------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| 1996      | [CSS 1 发布](https://www.w3.org/TR/CSS1/)                            | 选择器、字体、颜色、盒模型雏形                                                    |
| 1998      | [CSS 2 发布](https://www.w3.org/TR/2008/REC-CSS2-20080411/)          | 定位（`position`）、`float`、媒体类型——之后**十年没大动**                         |
| 1999-2009 | [IE6 时代](https://en.wikipedia.org/wiki/Internet_Explorer_6)        | 浏览器实现互不一致，前端时间一半花在写 hack 上；jQuery 因此火了                   |
| 2011      | [Flexbox 草案](https://www.w3.org/TR/2011/WD-css3-flexbox-20110322/) | 终于有了"主轴 / 交叉轴"心智                                                       |
| 2017      | [Flex / Grid 在主流浏览器全支持](https://caniuse.com/css-grid)       | **现代布局元年**——这是本章默认假设的浏览器基线                                    |
| 2020-2025 | [State of CSS 历年报告](https://stateofcss.com/)                     | 容器查询、`:has()`、原生嵌套、`@layer`、子网格、`color-mix()`、View Transitions ⋯ |

> 史料一手来源：W3C 各版规范是最权威的时间锚点；浏览器实际支持情况
> 看 [Can I Use](https://caniuse.com/)（按特性查"什么时候哪个浏览器
> 支持的"）；社区使用率看 [State of CSS](https://stateofcss.com/zh-Hans/)
> 历年问卷。想深入读 CSS 演进史可参考 Eric Meyer 的
> [CSS: The Definitive Guide](https://meyerweb.com/eric/books/css-tdg/)
> （O'Reilly，每版都更新）。

**两个结论值得记住**：

1. **2017 年是分水岭**——之前的"前端写 CSS 经验"很大一部分是在
   对抗浏览器差异；之后才真正回到"用 CSS 表达设计"。如果你在网上
   看到 2017 年之前的 CSS 教程，**先确认它讲的是不是现代写法**
2. **CSS 不是"简单"，是"被简化呈现"**——下面这些机制每一个都能写
   一篇长文。**它们不是冷知识，是你"为什么这个 div 不居中"的真正
   答案**：

   | 中文术语         | 英文全称（缩写）               | 一手参考                                                                                          |
   | ---------------- | ------------------------------ | ------------------------------------------------------------------------------------------------- |
   | 选择器优先级     | specificity                    | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_cascade/Specificity)                   |
   | 层叠             | cascade                        | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_cascade/Cascade)                       |
   | 继承             | inheritance                    | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_cascade/Inheritance)                   |
   | 包含块           | containing block               | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_display/Containing_block)              |
   | 堆叠上下文       | stacking context               | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_positioned_layout/Stacking_context)    |
   | 外边距折叠       | margin collapsing              | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_box_model/Mastering_margin_collapsing) |
   | 块级格式化上下文 | BFC = Block Formatting Context | [MDN](https://developer.mozilla.org/zh-CN/docs/Web/CSS/CSS_display/Block_formatting_context)      |

## 这一章和"我学过的 HTML / CSS"有什么不同

很多人以为 HTML / CSS 是"入门内容"，学过基础就够了。**这是 2026 年
最容易踩的认知坑**：CSS 在 2020-2025 年经历了**一次结构性升级**——
上面表格里 2020 后那一行的每一项都改写了既往的"最佳实践"。

但这一章不会只讲新语法——那样和 MDN / web.dev 没区别。我们的目标
是带你看穿三件事：

1. **现代写法的"前世"**：每一个新特性（容器查询、`auto-fit`、
   subgrid、`:has()`）都在替换某个 2010 年代的 hack。讲清楚替换了
   什么、为什么旧方案失败，你才会真正记住新方案
2. **"看似简单"下藏着的复杂面**：margin 折叠、BFC、堆叠上下文、
   包含块——业务里"为什么我的弹窗 z-index 9999 还是被盖住"
   "为什么父元素塌陷"全靠这些。本章会逐节点破，不藏
3. **AI 协作的边界**：CSS 是 AI 输出最容易"看起来对实际错"的领域
   之一。哪些任务直接交、哪些必须自己 review，每节都会标

> **写给已经写过几年前端的读者**：如果你最近一次系统过 CSS 是 2020
> 年之前，这一章你需要重读。不是因为旧知识错了，是因为**很多以前
> 必须靠 JS / 预处理器 / hack 解决的问题，现在 CSS 原生就能做**——
> 还在用旧办法的代码就是技术债。

> **写给新入行的读者**：你是赶上了好时候。这一章里讲的几乎全部都是
> "现代写法"——浮动布局、清除浮动、IE 兼容、CSS Hack、`@import`
> 嵌套技巧那一套**完全不必学**（详见 [SPEC-0003 §显式移除清单](https://github.com/wilsonwangdev/frontend-engineer-handbook/blob/main/specs/0003-table-of-contents/spec.md)）。
> 你直接学到现在还会用 5 年的写法。但**不要因此轻视 CSS**——
> 上面史前史里那些"复杂面"（cascade / 堆叠上下文 / margin 折叠等）
> 是真正的实战障碍，本章会反复点回来。

| 知识点                                                | 归位         | 理由                                                                                                                                     |
| ----------------------------------------------------- | ------------ | ---------------------------------------------------------------------------------------------------------------------------------------- |
| Flexbox 主轴 / 交叉轴心智                             | **必学**     | 90% 的"水平 / 垂直居中""平均分布"靠这个                                                                                                  |

| ARIA 属性                                             | **仍需理解** | 知道何时该用 `role` / `aria-label`；具体 attribute 让 AI 写 + review                                                                     |
| `:has()` / `:is()` / `:where()`                       | **必学**     | 写组件库 / 状态切换时大幅减少 JS                                                                                                         |

| CSS 嵌套（原生）                                      | **仍需理解** | 知道语法 + 知道与 SCSS 的差异；具体写法 AI 写得熟                                                                                        |

| CSS 预处理器（Sass / Less）详尽语法                   | **可委托**   | 现代项目用原生嵌套 + 变量已够；选 Sass 也让 AI 写                                                                                        |

## 3.3 怎么读这一章

### 别脱离实时预览读

CSS 与第 2 章的"协议 / 渲染 / 安全"不同——它**改一行代码就能立刻在
屏幕上看到结果**。所以这一章的读法是：

1. 打开 [CodePen](https://codepen.io) / [CSS Playground](https://playcode.io/css)
   或本地任何 HTML 文件
2. 一边读一边把例子敲进去
3. 改一两个值看看变化（`gap: 1rem` 改成 `2rem`、`grid-template-columns`
   改列数）

只读不写——基本等于没读。

### AI 的位置

| 你想做                                 | AI 帮得上吗                                  |
| -------------------------------------- | -------------------------------------------- |
| 写一段 Flex / Grid 居中                | ✅ 现代 AI 写得熟                            |
| 把"圣杯布局" / "三栏自适应"翻译成 Grid | ✅ 直接给                                    |
| 选 Flex 还是 Grid                      | ⚠️ 心智模型必须在你脑子里，AI 给的是默认答案 |
| 写 a11y 属性（ARIA）                   | ✅ 但你必须读懂——错配的 ARIA 比没有更糟      |
| 设计响应式断点                         | ❌ 业务上下文 AI 拿不到，让它列方案、你拍板  |
| 设计设计令牌（颜色 / 字号 / 间距体系） | ❌ 这是产品 / 视觉决策，不是技术决策         |

## 3.4 设计判断暗线

按 [SPEC-0013](https://github.com/wilsonwangdev/frontend-engineer-handbook/blob/main/specs/0013-design-principles-as-undercurrent/spec.md)
的暗线规则，本章会在以下位置点破设计判断：

- **§3.1 现代布局**：声明式（declarative）vs 命令式（imperative）——
  `display: grid` 是"我要什么"，旧式 JS 算坐标是"怎么做"；声明式让浏览器
  做调度，AI 也写得更稳
- **§3.3 @layer**：分层控制（layered specificity）——`@layer reset, base,
components, utilities` 把样式优先级从"谁的选择器更长"改成"谁在哪一层"，
  消除大型项目里"不停叠 `!important`"的恶性循环
- **§3.5 设计令牌**：信息隐藏（information hiding）——色值、字号、间距
  暴露成"语义名"（`--color-primary`）而非"具体值"（`#3b82f6`），换主题
  时不用全局搜索替换
- **§3.6 View Transitions**：声明式动画（declarative animation）——
  浏览器接管"前后状态差异 → 平滑过渡"的调度，告别旧式手写
  FLIP 计算
